# AI Papers Digest — 2026-04-28
Selected recent arXiv papers.

## Overview
The strongest recent arXiv papers focus less on scaling one monolithic model and more on making AI systems auditable, cheaper, safer, and more effective in realistic workflows. Researchers are tackling debugging for multi-agent systems, token economics for coding agents, long-horizon GUI and robotics control, and better evaluation for math and code generation. The overall pattern is that practical AI progress now depends heavily on systems design, measurement, and deployment discipline around the model.

## Papers
### 1. Seeing the Whole Elephant: A Benchmark for Failure Attribution in LLM-based Multi-Agent Systems
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22708
Summary: This paper introduces TraceElephant, a benchmark for identifying which agent and which step caused a failure in LLM-based multi-agent systems using full execution traces. That matters because debugging multi-agent stacks is becoming a major bottleneck as orchestration grows more complex and outputs become harder to interpret. A benchmark aligned with real developer-visible traces could make agent failures easier to diagnose and compare across systems.

### 2. SOLAR-RL: Semi-Online Long-horizon Assignment Reinforcement Learning
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22558
Summary: The authors propose a semi-online RL framework for multimodal GUI agents that reconstructs rollout candidates from static data and assigns rewards based on trajectory quality. This matters because long-horizon desktop and app automation is a core target for agentic AI, but pure online training is costly and unstable. If semi-online feedback can reliably improve completion rates, it offers a more practical path for training real GUI agents.

### 3. Thinking Without Words: Efficient Latent Reasoning with Abstract Chain-of-Thought
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22709
Summary: This work replaces long natural-language chains of thought with short abstract token sequences from a reserved vocabulary, cutting reasoning-token use while preserving performance. This matters because inference cost and latency are emerging as major constraints on reasoning-heavy systems. A latent reasoning method that keeps quality while shrinking token budgets could be valuable for production-grade reasoning and agent deployments.

### 4. How Do AI Agents Spend Your Money? Analyzing and Predicting Token Consumption in Agentic Coding Tasks
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22750
Summary: The paper studies token consumption across frontier models on agentic coding tasks and finds that these workflows are unusually expensive, highly variable, and often poorly predicted even by the models themselves. This matters because the economics of agents increasingly determine whether they are deployable at scale. Quantifying where tokens go and which models are more efficient can directly shape product design and model selection.

### 5. GazeVLA: Learning Human Intention for Robotic Manipulation
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22615
Summary: GazeVLA uses human gaze as an explicit proxy for intention, pretraining on egocentric human data before adapting to robotic manipulation tasks. This matters because embodied systems still struggle to transfer useful structure from human behavior into robot action. Modeling intention before action could improve generalization and robustness for long-horizon manipulation in both simulation and real-world settings.

### 6. RedVLA: Physical Red Teaming for Vision-Language-Action Models
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22591
Summary: RedVLA proposes a red-teaming framework for uncovering unsafe behaviors in vision-language-action models by synthesizing risky scenes and amplifying failures across models. This matters because VLA systems can cause irreversible physical harm, making pre-deployment safety testing much more demanding than for text-only models. A systematic way to surface safety failures could become an important part of embodied-AI evaluation pipelines.

### 7. RealBench: A Repo-Level Code Generation Benchmark Aligned with Real-World Software Development Practices
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22659
Summary: RealBench evaluates repository-level code generation using natural-language requirements plus UML-style design inputs, closer to how enterprise software is actually specified. This matters because many code benchmarks overstate real usefulness by ignoring structured design artifacts and repo-scale constraints. A more realistic benchmark can better reveal where coding models help and where they still fail in production development.

### 8. Rethinking Math Reasoning Evaluation: A Robust LLM-as-a-Judge Framework Beyond Symbolic Rigidity
Date: 2026-04-24
URL: https://arxiv.org/abs/2604.22597
Summary: This paper argues that strict symbolic answer checking is too brittle for modern math reasoning benchmarks and proposes an LLM-based judging framework that handles more varied valid outputs. This matters because evaluation noise can distort claims of progress on reasoning-heavy models. Better judging infrastructure improves both benchmarking reliability and the feedback loops used to train stronger mathematical reasoners.

## Themes
- Multi-agent systems need better observability, debugging, and cost accounting to become reliable in production.
- Long-horizon agent training is shifting toward methods that reuse offline data while still approximating richer feedback.
- Reasoning research is increasingly focused on reducing inference-token cost rather than only increasing depth.
- Embodied AI progress depends on both stronger intention modeling and stronger physical safety evaluation.
- Benchmark design is moving closer to real-world developer and user settings rather than simplified academic tasks.
