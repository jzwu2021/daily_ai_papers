# AI Papers Digest — 2026-05-01
Selected recent arXiv papers.

## Overview
The freshest arXiv papers continue the shift from isolated model gains toward the system-level ingredients needed to make AI more efficient, grounded, and deployable. The set spans new reasoning methods for small models, agent-development infrastructure, stronger multimodal spatial understanding, embodied planning, code-generation evaluation, and hardware-aware inference design. The common thread is that recent progress is coming from better interfaces between models, tools, environments, and serving stacks rather than from scaling alone.

## Papers
### 1. Turning the TIDE: Cross-Architecture Distillation for Diffusion Large Language Models
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26951v1
Summary: This paper proposes TIDE, a framework for distilling large diffusion language models into much smaller students even when teacher and student use different architectures, attention mechanisms, and tokenizers. That matters because diffusion LLMs look promising but are still expensive and cumbersome to deploy broadly. A workable cross-architecture distillation path could make these models more practical for code and general language tasks without requiring frontier-scale inference budgets.

### 2. Three-Step Nav: A Hierarchical Global-Local Planner for Zero-Shot Vision-and-Language Navigation
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26946v1
Summary: The authors propose a simple three-view procedure—look forward, look now, and look backward—to reduce drift and premature stopping in zero-shot vision-and-language navigation. This matters because long-horizon embodied agents often fail through cumulative planning errors rather than a single bad step. An add-on planner that improves zero-shot navigation without retraining could be useful for practical robotics and agentic spatial tasks.

### 3. Select to Think: Unlocking SLM Potential with Local Sufficiency
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26940v1
Summary: This work argues that small language models often already contain the large model’s preferred next token within their top-K candidates, and uses that insight to train an SLM to rerank its own proposals. This matters because efficient reasoning is increasingly limited by latency and inference cost, not just raw capability. If small models can recover much more reasoning performance through lightweight selection rather than full external generation, they become more viable for production use.

### 4. World2VLM: Distilling World Model Imagination into VLMs for Dynamic Spatial Reasoning
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26934v1
Summary: World2VLM uses a generative world model to synthesize future views under camera motion, then distills that spatial imagination into a vision-language model during training. This matters because VLMs remain much better at static understanding than at reasoning about how scenes evolve over time. Teaching a VLM this capability at training time, instead of bolting on a world model at inference time, could improve spatial reasoning with lower deployment overhead.

### 5. ClassEval-Pro: A Cross-Domain Benchmark for Class-Level Code Generation
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26923v1
Summary: ClassEval-Pro introduces 300 class-level code-generation tasks across 11 domains and shows that even strong frontier models still struggle badly on this middle ground between single functions and full repositories. This matters because class construction is a common unit of real software work, yet it has been under-measured compared with toy function benchmarks. A stronger benchmark here should make it easier to see whether coding models are actually improving on production-relevant software structure.

### 6. ClawGym: A Scalable Framework for Building Effective Claw Agents
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26904v1
Summary: ClawGym presents a full framework for developing personal agents that operate over files, tools, and persistent workspace state, including synthetic task generation, benchmark construction, and rollout-based training. This matters because agent progress is often bottlenecked by the surrounding environment and evaluation loop rather than by the base model alone. A reusable framework for verifiable task generation and diagnostics could accelerate more disciplined agent development.

### 7. STARRY: Spatial-Temporal Action-Centric World Modeling for Robotic Manipulation
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26848v1
Summary: STARRY aligns spatial-temporal prediction with action generation for robotic manipulation, using predicted depth and end-effector geometry to modulate action attention. This matters because robot policies often struggle when future interaction structure is important but weakly represented in the policy itself. Better action-centric world modeling could improve manipulation robustness in both simulation and real-world settings.

### 8. Exploring the Efficiency of 3D-Stacked AI Chip Architecture for LLM Inference with Voxel
Date: 2026-04-29
URL: https://arxiv.org/abs/2604.26821v1
Summary: This paper introduces Voxel, a compiler-aware end-to-end simulation framework for analyzing how 3D-stacked AI chips behave on LLM inference workloads. This matters because the memory bottleneck remains one of the biggest obstacles to efficient large-model serving. A tool that exposes how compiler choices, mappings, bandwidth, topology, and thermal limits interact could help the ecosystem design better inference hardware instead of optimizing one layer in isolation.

## Themes
- Efficient AI is increasingly about distillation, serving, and hardware design rather than only about larger base models.
- Agent progress depends heavily on better environments, benchmarks, and development frameworks.
- Multimodal and embodied systems are improving by learning richer spatial and temporal structure.
- Code-generation research is moving toward more realistic software units and evaluation settings.
- Practical deployment pressure is pushing research toward lower-overhead reasoning and more predictable inference stacks.
